# 한인마트 취업 대비 한국어 학습 웹앱 — 설계 문서

- 날짜: 2026-07-13
- 상태: 사용자 승인됨 (설계 대화에서 "이대로 진행" 확인)

## 1. 목적과 학습자

필리핀에서 한인마트(H마트류) 취업을 준비하는 필리핀인 친구를 위한 한국어 학습 앱.

- 학습자: 영어 능통, K콘텐츠로 귀만 트인 완전 초보 (한글 못 읽음)
- 최우선 목표: 한국인 고객 응대에 필요한 **스피킹 & 리스닝**
- 부차 목표: 상품 분류·매장 라벨을 읽는 **기초 리딩**
- UI 언어: 영어 (친구가 혼자 학습 가능해야 함)
- 사용 기기: 아이패드 (사파리), 터치 중심

## 2. 콘텐츠 구성 — 5개 탭

### 🔤 Sounds (소리 기초)
영어에 없는 한국어 소리만 다루는 짧은 크래시 코스 (6개 항목):
1. ㅓ = "uh" (어서 오세요의 어)
2. ㅡ = "eu" (입술 안 둥글게)
3. ㅐ/ㅔ = "eh"
4. 된소리 ㄲ/ㄸ/ㅃ/ㅆ/ㅉ (숨 없이 강하게 — 떡 tteok)
5. ㅈ/ㅊ (j/ch)
6. 받침(끝소리) — 밥, 김, 물처럼 끊기는 소리

각 항목: 설명(영어) + 마트 단어 예시 + 음성 재생.

### 💬 Phrases (응대 표현) — 시나리오 레슨 6개 + 숫자 미니레슨, 총 약 80문장
1. **Greeting & Offering Help** — 어서 오세요 / 안녕하세요 / 찾으시는 거 있으세요? / 천천히 보세요
2. **Understanding Customers (리스닝 전용)** — 손님이 하는 말 알아듣기: 라면 어디 있어요? / 이거 얼마예요? / 덜 매운 거 있어요? / 이거 언제까지예요? / 봉투 하나 주세요
3. **Directing & Locating** — 이쪽으로 오세요 / 3번 통로에 있어요 / 냉동 코너에 있어요 / 저기 있어요 / 제가 안내해 드릴게요
4. **Price & Checkout** — 봉투 필요하세요? / 카드 되세요 / 현금이요, 카드요? / 영수증 드릴까요? / 감사합니다
5. **Stock & Problems** — 죄송합니다, 품절이에요 / 내일 들어와요 / 확인해 볼게요 / 교환해 드릴게요 / 잠시만 기다려 주세요
6. **Small Talk & Thanks** — 감사합니다, 또 오세요 / 안녕히 가세요 / 좋은 하루 되세요 / 오랜만이에요
7. **Numbers 미니레슨** — 손님이 말하는 가격·수량 알아듣기: 한자어 수(일이삼…천/만), 고유어 수+개(한 개, 두 개), 듣기 예시 문장

각 문장 카드: 한글(크게) + respelling + 영어 뜻 + 상황 메모 + 음성(보통/느리게).
레슨 2는 "듣고 이해"가 목표임을 명시 (따라 말하기 아님).

### 🛒 Products (상품 카탈로그) — 8개 카테고리, 약 90개 상품
카테고리: ① 라면·면류 ② 장류·소스(고추장/된장/쌈장/간장/참기름) ③ 김치·반찬 ④ 냉동(만두/떡/어묵) ⑤ 과자·빵 ⑥ 음료·주류(소주/막걸리/식혜) ⑦ 쌀·곡물·가루 ⑧ 생활용품·기타

각 상품 카드:
- 한글 표기 (크게)
- 포장지 실제 영문 표기 (예: Gochujang, Tteokbokki) — 매장 실물 인식용
- 발음 respelling
- 영어 한 줄 설명 (예: "spicy fermented red pepper paste")
- 관련 손님 질문 예문 연결 (예: 고추장 어디 있어요?)

### ㄱ Hangul (한글 미니코스) — 4단계
1. 기본 자음 + 기본 모음 — 상품 단어로 배우기 (라면 → ㄹ+ㅏ / ㅁ+ㅕ+ㄴ)
2. 나머지 자모 + 음절 블록 조립 원리
3. 받침 읽기
4. **매장 라벨 필수 단어**: 매운맛/순한맛, 냉동/냉장, 유통기한, 세일/신상품/증정, 소/중/대

목표: 상품명과 라벨을 더듬더듬 읽는 수준. 각 글자·단어 탭 시 음성.

### 🎯 Quiz (퀴즈) — 3모드
1. **Listening**: 음성 듣고 4지선다로 뜻 고르기. "손님 말 알아듣기" 세트 포함
2. **Speaking**: 영어 상황 제시 → 스스로 한국어로 말해보기 → 탭하면 정답 문장+음성 공개 → 자기평가(맞음/틀림)
3. **Label Reading**: 한글 상품명/라벨 보고 뜻 고르기

퀴즈는 탭별 학습 데이터에서 무작위 출제. 점수는 세션 내 표시.

## 3. 발음 표기 규칙

- 주 표기: **영어 파닉스 기반 respelling**, 음절을 하이픈으로 구분 (어서 오세요 → "uh-suh oh-seh-yo")
- 표준 로마자 표기법(eo/eu)은 영어 화자가 오독하므로 문장에는 쓰지 않음
- 예외: 상품명은 포장지에 실제 인쇄되는 통용 표기(gochujang, ramyun 등)를 **병기**
- 연음·비음화 등 실제 소리 나는 대로 respelling (합니다 → hahm-nee-da)
- 모든 한국어 텍스트는 탭 → TTS 재생, 느리게(0.7x) 버튼 제공

## 4. 기술 구조

- **단일 자체완결 HTML 파일** (`index.html`) — CSS/JS/데이터 전부 내장, 외부 요청 없음, 오프라인 동작
- 콘텐츠 데이터는 JS 객체(레슨/상품/한글/퀴즈 공용)로 한 곳에 정의 — 퀴즈는 같은 데이터에서 생성
- 음성: Web Speech API `speechSynthesis`, `lang: 'ko-KR'` — 아이패드 내장 한국어 음성(유나) 사용, 인터넷 불필요
- 진행 상태: localStorage (카드별 "외웠음" 체크, 마지막 위치)
- 반응형·터치 최적화 (아이패드 세로/가로, 폰에서도 동작), 라이트/다크 테마 대응
- 전달 경로 2가지: ① Artifact로 게시해 링크 공유 ② HTML 파일 자체를 에어드랍/메신저로 전송 (둘 다 동일하게 동작)

### PDF 다운로드 버튼
- 치트시트 PDF (전 문장+발음+뜻 표, 상품표, 한글표, 인쇄해서 계산대 옆에 두는 용도)
- 1안: 빌드 시점에 PDF를 미리 생성해 base64로 앱에 내장 → 버튼 탭 시 Blob으로 저장/공유 시트
- 폴백: 헤드리스 브라우저 등 PDF 생성 수단이 없으면 인쇄 전용 뷰 + `window.print()` (아이패드에서 프린트 → PDF 저장 가능)
- 어느 쪽이든 사용자에겐 "Download PDF" 버튼 하나로 보임

## 5. 제약·에러 처리

- 아이패드 TTS는 사용자 제스처 후에만 재생 가능 → 시작 화면에 "Tap to enable sound" 버튼, 첫 탭에서 무음 발화로 활성화
- 한국어 음성이 없는 기기: 감지 후 상단에 안내 배너 (respelling으로 학습은 계속 가능)
- localStorage 불가 환경(프라이빗 모드 등): 진행 저장만 조용히 비활성화, 학습 기능은 정상
- TTS 속도·음성 품질은 기기 의존 — 느리게 버튼으로 보완

## 6. 테스트 계획

- 맥 사파리/크롬에서 TTS(ko-KR) 재생, 느리게 재생 확인
- 퀴즈 3모드 출제·채점 로직 확인 (데이터 무작위 추출, 오답 보기 중복 없음)
- PDF 버튼: 다운로드 동작 + 내용물(한글 폰트 렌더링) 확인
- 오프라인 확인: 로컬 파일로 열어 전 기능 동작
- 아이패드 실기기 확인은 사용자(루시)가 최종 수행

## 7. 범위 제외 (v1에서 안 함)

- 음성 인식(발음 채점) — Web Speech 인식은 iOS 사파리에서 불안정
- 서버/계정/동기화 — localStorage로 충분
- 이미지/사진 — 파일 크기와 저작권 문제, 텍스트+이모지로 대체
- 간격 반복(SRS) 알고리즘 — v1은 단순 "외웠음" 체크
